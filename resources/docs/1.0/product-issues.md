# Product Issues

The **Product Issues** page is used to review product issues found during the purchase, receiving, or warehouse process. It helps the user identify shortage, excess, vendor-related, and warehouse-related product issues.

---

## Open Product Issues

![Product Issues](/docsimages/pi1.png)

To open the Product Issues page:

1. Open the main menu.
2. Click **Product Issues**.
3. The system will show the **Product Issues** page.

The Product Issues page is available from the `/product-issues` URL.

---

## View Product Issues

![Product Issues List](/docsimages/pi2.png)

The **Products Table** shows all product issue records.

The table includes the following details:

1. **Purchase Order Id**
2. **Brand Title**
3. **SKU Code**
4. **MRP**
5. **GST**
6. **HSN**
7. **Quantity Requirement**
8. **Available Quantity**
9. **Purchase Rate**
10. **Received Quantity**
11. **Issue From**
12. **Issue**
13. **Issue Items**
14. **Issue Reason**

The purchase order number is clickable and opens the related purchase order details.

---

## Filter Product Issues

![Product Issues Filter](/docsimages/pi3.png)

The Product Issues page provides tabs to filter issue records.

Available tabs:

1. **All** shows all product issue records.
2. **Vendor Issues** shows issues created from the vendor side.
3. **Warehouse Issues** shows issues created from the warehouse side.

Click a tab to filter the table based on the issue source.

---

## Issue Details

Each product issue row shows the reason and quantity connected to the issue.

Use the following fields to understand the issue:

1. **Issue From** shows whether the issue is from Vendor or Warehouse.
2. **Issue** shows the issue type.
3. **Issue Items** shows the affected quantity.
4. **Issue Reason** shows the detailed description of the issue.

---

## Summary

The Product Issues process follows these steps:

1. Open **Product Issues** from the menu.
2. Review all product issue records in the table.
3. Use **Vendor Issues** to view vendor-side issues.
4. Use **Warehouse Issues** to view warehouse-side issues.
5. Open the purchase order link if the related purchase order needs to be checked.
6. Review the issue type, issue quantity, and issue reason.
